# CodeGraph — Local AI Memory Layer for Codebases

Local-first MVP. CLI + minimal web UI + MCP server. See [README.md](README.md) for user docs.

## Stack (LOCKED — do not relitigate per session)

Python 3.11, **uv** for package mgmt, **tree-sitter-languages** for parsing, **DuckDB** for graph + vector storage, **sentence-transformers** (`all-MiniLM-L6-v2`) for embeddings, **anthropic** SDK (`claude-sonnet-4-6`) for GraphRAG, **FastAPI + uvicorn** for HTTP, **typer + rich** for CLI, **React + Vite + D3** for web UI, **mcp** SDK for MCP server.

Full rationale in [BUILD_PLAN.md](BUILD_PLAN.md) §1.

## Resume protocol (do this first in every session)

1. Read [STATUS.md](STATUS.md) — current phase + next atomic task.
2. Read `plan/<NN>-*.md` for the current phase — task definitions (files, verify cmd, commit msg). Each plan file is 1–8KB and self-contained. **Do NOT load BUILD_PLAN.md** unless you need full rationale.
3. Run `uv run pytest -x` to confirm green baseline.
4. Execute exactly that one task — no scope creep.
5. Update [STATUS.md](STATUS.md): mark task done, set next task, note blockers.
6. Commit: message MUST start with task ID (e.g. `T2.4: ...`).
7. `git push` to origin/main.

## Conventions

- **One atomic task = one commit.** `git log --oneline` is the progress log.
- **Push after every commit.** `git push` once per task — the GitHub repo (`origin/main`) must stay current so the work feels active and CI runs on every change. Never let local commits accumulate unpushed.
- **No AI attribution anywhere in git.** STRICT RULE. Never add `Co-Authored-By` lines, "Generated by …" tags, or mentions of any AI agent / coding assistant in commit messages, PR descriptions, code comments, docs, or file headers. The repo belongs to the human author. Treat git history as if a solo developer wrote every line. The `anthropic` SDK and `claude-sonnet-4-6` model are allowed since they are dependency / API identifiers — package and model names, not attributions.
- **Never break the CLI on main.** `uv run codegraph --help` must always work after T0.4.
- **Verify before commit:** `uv run ruff check && uv run pytest`.
- **No new dependencies without updating §1 of BUILD_PLAN.md.**
- **No new top-level packages.** Stick to the layout in BUILD_PLAN.md §2.
- All long-lived MD trackers (AGENTS.md, STATUS.md, BUILD_PLAN.md, README.md, source spec) live at **repo root** for easy updating.

## If something seems wrong

- Compare against UIR schema and DuckDB schema first — they are source of truth.
- Check `git log --oneline` for prior task implementation patterns.
- DO NOT add features outside current task scope. File a note in STATUS.md "Future:" section.
- If a dep version conflicts with the plan, fix it in pyproject.toml and note the deviation in STATUS.md.
