# CodeGraph — Agent Conventions

Local-first MVP. CLI + minimal web UI + MCP server. See [README.md](README.md) for user
docs and [STATUS.md](STATUS.md) for current state and next tasks.

## Stack (locked — do not relitigate per session)

Python 3.11, **uv** for package mgmt, **tree-sitter-languages** for parsing, **DuckDB** for
graph + vector storage, **sentence-transformers** (`all-MiniLM-L6-v2`) for embeddings,
**anthropic** SDK (`claude-sonnet-4-6`) for GraphRAG, **FastAPI + uvicorn** for HTTP,
**typer + rich** for CLI, **React + Vite + D3** for web UI, **mcp** SDK for MCP server.

## Start here

1. Read [STATUS.md](STATUS.md) — `## Current` has status, test count, and next tasks.
2. Run `uv run pytest -q` (or `./.venv/Scripts/python.exe -m pytest -q` if `uv run` can't
   resolve the venv) to confirm a green baseline before changing anything.
3. Work the task at hand — no unrelated scope creep in the same change.
4. Update [STATUS.md](STATUS.md) if the task changes status, test count, or next steps.

## Conventions

- **Push after every commit.** `git push` once per commit — `origin/main` should stay
  current so CI runs on every change. Don't leave commits unpushed.
- **No AI attribution anywhere in git.** STRICT. Never add `Co-Authored-By` lines,
  "Generated by …" tags, or mentions of any AI agent / coding assistant in commit
  messages, PR descriptions, code comments, docs, or file headers. Treat git history as
  if a solo developer wrote every line. The `anthropic` SDK and `claude-sonnet-4-6` model
  name are fine — they're dependency/API identifiers, not attributions.
- **Commit messages:** conventional-commit style (`fix:`, `feat:`, `docs:`, `chore:`,
  `test:`) with a short imperative summary — matches `git log --oneline`.
- **Never break the CLI on main.** `uv run codegraph --version` (the CI smoke check) must
  always pass.
- **Verify before commit:** `uv run ruff check && uv run ruff format --check && uv run
  pytest -q` all green. Don't commit with failing tests or lint errors.
- **Comments explain *why*, not *what*.** Identifiers already say what; a comment earns
  its place only when the reason is non-obvious — a workaround, a hidden constraint, a
  subtle invariant. No paraphrasing the code.
- **No half-finished features on main.** `main` is always shippable.
- **Fetch before push.** `git fetch origin && git log --oneline HEAD..origin/main` before
  pushing, to catch remote divergence early.

## If something seems wrong

- Compare against the UIR schema and DuckDB schema first — they're the source of truth.
- Check `git log --oneline` for prior implementation patterns on similar work.
- Don't add features outside the current task's scope — note it in STATUS.md instead.

## End-of-session checklist

1. All edited files saved.
2. `uv run ruff check && uv run ruff format --check && uv run pytest -q` green.
3. STATUS.md updated if status/tests/next-steps changed.
4. Commit with a conventional-commit message — no AI attribution.
5. `git push` — never leave commits unpushed.
